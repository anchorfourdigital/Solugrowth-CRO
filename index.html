Summary

The offer is credible. The page isn't selling it.

SoluGrowth has the ingredients of a credible B2B outsourcing provider — a broad service portfolio, enterprise-level client names, and a genuine cost-arbitrage story. The site consistently fails to convert that credibility into enquiries. The hero doesn't answer the visitor's first question fast enough, calls to action compete with each other instead of pointing to one action, trust signals are present but under-leveraged, and the same weak-CTA pattern repeats across every template — which is good news, because it means the fix is structural, not page-by-page.

A note on the data behind this report: This audit combines a manual page-by-page review with a structural scan run through AnchorFour Digital's internal CRO auditing tool. The client's actual GA4 traffic and conversion data was not available at the time of this audit — the scan was run using placeholder inputs purely to surface structural and UX patterns. Every finding below is a direct observation about the page itself. No lead-volume or revenue projections are included in this report, since those numbers would rest on data we don't yet have. We'd recommend requesting GA4 access as a next step so future reporting can quantify impact with real numbers.

Root cause

Total page weight is approximately 18MB, and image assets alone account for an estimated 16MB+ of recoverable savings (compression, resizing, modern formats like WebP/AVIF). This single fix is very likely responsible for most of the gap between a fast First Contentful Paint (0.4–1.7s — something starts rendering quickly) and a slow Largest Contentful Paint and Speed Index (the actual page taking 3–30 seconds to finish rendering, worse on mobile connections). Supporting Lighthouse insights: inefficient cache lifetimes (6.5–6.7MB recoverable), render-blocking requests (1.8–2.8s recoverable), unused JavaScript (667–686KB) and unused CSS (~107KB).

Site does not use HTTPS — mixed content on every page load

High
Site-wide — 11 insecure requests (desktop), 9 insecure requests (mobile)

Lighthouse flags the site for serving insecure (non-HTTPS) requests on both desktop and mobile. For a company whose core pitch is trusted handling of clients' financial and accounting data, this is more than a technical checkbox — modern browsers surface mixed-content warnings directly to visitors, which directly undermines the trust story the site is trying to tell. This is the single largest contributor to the Best Practices score sitting at 50/100 on both device types.

Fix

Audit all page resources (images, scripts, embeds) for hardcoded http:// references and update to https://, then confirm the site enforces HTTPS site-wide with a redirect and updated HSTS policy.

High Priority

7 findings

Hero doesn't answer "why should I care" fast enough

High
Homepage — above the fold

The hero is a rotating slider stacking several unrelated headlines ("SME Accounting," "ISO recertification," "IT Outsourcing," "Navigating Solutions") with no single, quantified value proposition visible on load. A visitor evaluating a BPO provider applies a fast filter — what do I get, roughly what does it save me, and why this provider — and the current hero doesn't answer any of the three before the slide changes.

Fix

Replace the slider with one fixed, outcome-led headline plus a risk-addressing sub-headline (delivery timeline, compliance, or client count). Move the rotating messages into a secondary section further down the page rather than the first thing visitors see.

No single primary call to action

High
Header, hero, mid-page, footer, service sub-pages

The primary homepage CTA ("Learn more about our BPO services") is a plain text link, not a button, and every section — including every service sub-page we reviewed — offers its own differently-worded action instead of one consistent, high-value path. More options at each step means a lower chance any single one gets clicked.

Lighthouse's SEO audit confirms this at the link-text level: 4 links on the homepage carry no descriptive text — three "READ MORE" links on the Insights blog cards, and one "SEE MORE" link out to the Insights page. Beyond the CRO cost, this also means search engines and screen readers can't tell what any of these links actually lead to.

Fix

Pick one primary action across the whole site (e.g. a cost-saving assessment or discovery call) and repeat it — same wording, same button styling — in the hero, after each service block, and in the footer. Downgrade every other link to plain text. Separately, replace generic "READ MORE" / "SEE MORE" labels with descriptive text (e.g. "Read: Why SMEs Outsource Bookkeeping").

Confirmed manually + via AnchorFour CRO auditing tool + Lighthouse SEO audit

Contact form fields are inconsistent across the site

High
Homepage embedded form, /contact-us/, popup forms

The homepage and dedicated contact page use a short Name / Phone / Email / Message form, while the "Get in Touch" and cost-calculator popups ask for company size and more granular detail. Visitors converting through different entry points get a different experience with no clear reason why.

Fix

Standardise on one short form (name, work email, company, one dropdown for service interest) for the primary conversion path, and reserve the longer, detailed form for the cost calculator specifically, where extra detail is expected and justified.

Social proof is present but not specific

High
Client logo strip, near footer

Recognisable client logos (Absa, Nissan, Heineken, Barloworld, BCX) are a real trust asset, but they sit near the bottom of the page after a visitor has likely already decided to leave, and there are no named case studies or quantified outcomes to back them up — for a US buyer evaluating an offshore provider they've never met, unnamed or generic testimonials don't do the trust-building work that specific proof does.

Fix

Pull 3–5 logos into a strip directly beneath the hero. Pair them with 2–3 short, specific outcome quotes (named client where possible, or industry + size descriptor if NDA restrictions apply) rather than generic praise.

The South Africa delivery model isn't positioned as an advantage

High
About / value proposition section

Every US visitor evaluating this site is silently asking "why South Africa, and not India, the Philippines, or a domestic provider?" The site doesn't answer that objection directly anywhere we reviewed — genuine advantages like English-first language, partial EST time-zone overlap, and accounting-standards familiarity are implied by the site's general copy but never surfaced as a direct comparison. Unaddressed objections don't get raised as questions; they get resolved by the visitor leaving.

Fix

Add a dedicated comparison section — South Africa vs. India vs. US domestic — covering time zone, language, cost, and regulatory familiarity (confirm the specific accounting standards/certifications to cite before publishing). A simple table format reads as objective and builds trust faster than prose claims.

Surfaced via AnchorFour's CRO auditing tool

Header phone number is inconsistent and the tel: link is broken

High
Site-wide header — every page reviewed

The header displays "+1 636 542 0055" in one spot and "+1 636 542 005" (missing a digit) in another, and the underlying tel: link uses the shorter, incorrect number. This is identical across all five pages reviewed, confirming it's a global template issue rather than a one-off typo — meaning it's a one-fix problem, but currently every mobile visitor tapping to call is hitting a dead number.

Fix

Correct the number once in the global header/footer template and verify the tel: href matches the displayed number exactly.

Mobile visitors wait up to 30 seconds for the page to finish rendering

High
Homepage — confirmed via Lighthouse mobile audit

Largest Contentful Paint on mobile is 12.8 seconds (vs. 2.9s on desktop) and Speed Index is 30.2 seconds (vs. 8.4s on desktop). This isn't a projection — it's a measured result. Every CRO issue already flagged in this report (weak hero, buried calculator, scattered CTAs) is compounded by visitors on mobile connections potentially abandoning before the page has even finished rendering. This is the single highest-leverage technical fix in this report: it's caused by one thing (unoptimised images) and it affects every other finding's ability to even be seen.

Fix

See the Root Cause note in the Site Performance section above — image optimisation is the fix, and it's the same fix for both desktop and mobile.

Confirmed via Lighthouse mobile audit

Medium Priority

7 findings

"Why Outsource" page is written for one audience but linked as if for all

Medium
/why-outsource/

The page is linked from the main nav as a general case for outsourcing across all seven service lines, but the copy is written entirely around bookkeeping and finance ("in-house bookkeeper," "financial statements"). A visitor interested in staffing or customer care outsourcing lands on messaging that doesn't speak to them.

Fix

Either broaden the copy to speak to all service lines generically, with links out to service-specific "why outsource" sub-sections, or rename the nav link to reflect its actual finance/bookkeeping focus.

Cost-saving calculator is buried near the footer

Medium
Homepage — appears after client and partner logos

The calculator is a strong conversion lever — it offers a personalised value estimate rather than a generic contact form — but it sits below the client logos and partner strip, near the very end of the page, where fewer visitors will scroll to reach it.

Fix

Surface the calculator (or a link to it) much earlier — directly after the hero or value proposition section — so it's positioned as a primary engagement path, not an afterthought.

Service tiles describe features, not outcomes

Medium
Services page and sub-pages

Services are listed by name with a short description ("Bookkeeping & Accounting Services") rather than tied to a specific buyer outcome or metric. In a competitive evaluation, a catalogue of service names looks the same as every competitor's page.

Fix

Rewrite each tile as: service name → specific outcome → proof or metric. E.g. "Bookkeeping & Accounting — monthly close delivered in 3 days, GAAP-trained accountant, meaningful cost saving vs. an in-house hire."

Full navigation is exposed before visitors convert

Medium
Header navigation, all pages

The header nav exposes service sub-pages, blog, careers, and about links before a visitor has engaged with the core offer — every nav click is a potential exit from the conversion path.

Fix

Consider a reduced-navigation variant for the homepage specifically: logo, anchor links to on-page sections (Services, Why SoluGrowth, Contact), keeping Careers and Blog for a secondary "More" menu or footer only.

Surfaced via AnchorFour's CRO auditing tool

Copy reads keyword-stuffed in places

Medium
Homepage hero paragraph, Why Outsource intro

Phrases like "business process outsourcing BPO and BPO solutions... BPO services provider in business process outsourcing" repeat the same terms in ways that read as written for search engines rather than people, which undercuts the credibility the page is trying to build.

Fix

Rewrite for natural readability first; keep target terms present but not repeated within the same sentence or paragraph more than once.

FAQ content isn't confirmed to carry schema markup

Medium
Services page, Why Outsource page

Both pages carry well-structured FAQ sections, which is good practice, but this repeats across templates — meaning if FAQPage schema isn't implemented, it's a consistent missed opportunity for extra search result real estate rather than a single-page fix.

Fix

Confirm with a rich-results test whether FAQPage schema is present; if not, apply it at the template level so every page with an FAQ block benefits at once.

No reason to act now rather than later

Medium
Page-wide

Nothing on the pages we reviewed gives a visitor a reason to enquire today instead of bookmarking the page for "later" — which, in B2B, often means never.

Fix

A credible, non-manipulative prompt works well here — e.g. framing limited monthly onboarding capacity, or a time-bound incentive for booking an assessment this month.

Recommended Sequence

Where to start

  1. Optimise images site-wide and fix HTTPS mixed content — the single highest-leverage fix. Image compression alone recovers ~16MB per page load and is very likely the main driver of the 12.8s mobile Largest Contentful Paint; fixing insecure requests addresses both the Best Practices score and the trust story directly.
  2. Fix the header phone number and CTA hierarchy — both are template-level issues, so one fix resolves them site-wide.
  3. Rewrite the hero and pull the calculator + client logos higher up the homepage — this addresses the biggest above-the-fold drop-off risk, and will convert better once the page also loads faster.
  4. Standardise the contact form fields across the homepage, contact page, and popups.
  5. Request GA4 access from the client so the next round of reporting can quantify impact with real traffic and conversion data instead of structural observations alone.
